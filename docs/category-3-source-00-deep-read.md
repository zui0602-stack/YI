## 架構映射

- JS 基礎：17748 是後續 `zdjl.getVar`、`zdjl.setVar`、陣列、條件、循環、函式封裝的入門層。

- 後端與群控：17705 可抽象成設備心跳、命令輪詢、狀態表、超時告警、任務編號，不直接公開濫用控制面。
- 安全邊界：12789、6022 都提醒公開庫不能保留弱加密當安全方案，也不能提供未授權遠端取檔流程。
- 檢測與等待：138 補進視覺辨識策略，重點是縮小檢測範圍、避免長時間大範圍找圖。

## 萬龍腳本可用規則

- 每台紅手指/模擬器都要有 `device_id`、`role_name`、`task_state`、`last_seen_at`，方便群控與故障排查。
- 採集、行軍、回城、找資源都應寫成狀態字串，定期上報，不只靠畫面。
- 等待畫面不要只用全屏找圖；優先 OCR 關鍵字、固定 ROI、已知按鈕區域、狀態旗標。
- 隨機等待或重試次數要明確上下限，例如 `min <= n < max` 或 `min <= n <= max`。
- 任何遠端指令都要限制命令白名單，只允許安全任務代號，不允許任意文件路徑或任意 JS。
