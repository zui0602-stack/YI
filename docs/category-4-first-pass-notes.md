## 可抽象成實作模式

### 变量持久化

使用场景：

- UI 设置里的“记住值”不能覆盖所有动态写入场景。
- 脚本重启后仍要读取上次运行状态、坐标、选项或账号状态。

模式：

- 临时变量：动作内或脚本运行期使用。
- 设置变量并记住：适合用户手动配置。
- 本地存储：适合 JS 动态写入并在下次运行读取。

### 文件与图片变量

使用场景：

- 保存截图、图像变量或日志。
- 从 txt 中读取指定行作为变量。
- 把 OCR/识别/运行结果写入本地记录。

模式：

- 文本：读取文件后按换行拆分，用索引取行。
- 日志：时间戳 + 追加写入。
- 图片：识别屏幕后取得 base64，再转成二进制写入图片文件。

### 多条件与状态机

使用场景：

- 多个颜色/文字/图片条件同时成立才点击。
- 动作失败后回到第一步。
- 多个任务按不同时间间隔各自运行。

模式：

- 多颜色：封装检查函数，颜色点位和阈值存变量。
- 失败跳转：监听动作的运行失败，再控制跳转。
- 定时状态：每个任务记录下一次可运行时间，主循环只判断是否到时。
